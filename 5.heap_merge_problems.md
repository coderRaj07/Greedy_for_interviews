# 5️⃣ Heap Merge Problems (K-Way Merge Pattern)

Core thinking:

You have **K sorted lists**.

Instead of merging one by one:

```
L1 + L2 + L3 + ... + Lk
```

We use a **min heap** to always pick the **smallest among K candidates**.

---

## Core Template

```python
heap = []

push first element of each list

while heap not empty:

    pop smallest

    add next element from that list
```

Time Complexity

```
O(N log K)
```

where

```
N = total elements
K = number of lists
```

---

# 1️⃣ Merge K Sorted Lists

🔗 [https://leetcode.com/problems/merge-k-sorted-lists/](https://leetcode.com/problems/merge-k-sorted-lists/)

Given **k sorted linked lists**, merge into **one sorted list**.

---

### Visual

```
List1: 1 → 4 → 5
List2: 1 → 3 → 4
List3: 2 → 6
```

Heap initially:

```
1(L1) 1(L2) 2(L3)
```

Pop smallest repeatedly.

---

### Python

```python
import heapq

class Solution:
    def mergeKLists(self, lists):

        heap=[]

        for i,node in enumerate(lists):
            if node:
                heapq.heappush(heap,(node.val,i,node))

        dummy=ListNode(0)
        curr=dummy

        while heap:

            val,i,node=heapq.heappop(heap)

            curr.next=node
            curr=curr.next

            if node.next:
                heapq.heappush(heap,(node.next.val,i,node.next))

        return dummy.next
```

---

### Java

```java
class Solution {

    public ListNode mergeKLists(ListNode[] lists) {

        PriorityQueue<ListNode> pq =
            new PriorityQueue<>((a,b)->a.val-b.val);

        for(ListNode node:lists)
            if(node!=null)
                pq.add(node);

        ListNode dummy=new ListNode(0);
        ListNode curr=dummy;

        while(!pq.isEmpty()){

            ListNode node=pq.poll();

            curr.next=node;
            curr=curr.next;

            if(node.next!=null)
                pq.add(node.next);
        }

        return dummy.next;
    }
}
```

# 2️⃣ Merge K Sorted Arrays

🔗 [https://www.geeksforgeeks.org/problems/merge-k-sorted-arrays](https://www.geeksforgeeks.org/problems/merge-k-sorted-arrays)

Example

```
[1,4,7]
[2,5,8]
[3,6,9]
```

Result

```
[1,2,3,4,5,6,7,8,9]
```

---

### Python

```python
import heapq

def mergeKArrays(arr,k):

    heap=[]

    for i in range(k):
        heapq.heappush(heap,(arr[i][0],i,0))

    res=[]

    while heap:

        val,row,col=heapq.heappop(heap)

        res.append(val)

        if col+1 < len(arr[row]):
            heapq.heappush(heap,(arr[row][col+1],row,col+1))

    return res
```

# 3️⃣ Smallest Range Covering K Lists

🔗 [https://leetcode.com/problems/smallest-range-covering-elements-from-k-lists/](https://leetcode.com/problems/smallest-range-covering-elements-from-k-lists/)

Given:

```
List1: [4,10,15]
List2: [1,3,20]
List3: [5,18,22]
```

Find smallest range covering **at least one number from each list**.

---

### Greedy Idea

Maintain:

```
min element from heap
max element seen
```

Range =

```
max - min
```

Try to minimize.

---

### Python

```python
import heapq

class Solution:

    def smallestRange(self, nums):

        heap=[]
        max_val=float('-inf')

        for i,lst in enumerate(nums):

            heapq.heappush(heap,(lst[0],i,0))
            max_val=max(max_val,lst[0])

        best=[-10**5,10**5]

        while True:

            min_val,row,col=heapq.heappop(heap)

            if max_val-min_val < best[1]-best[0]:
                best=[min_val,max_val]

            if col+1 == len(nums[row]):
                break

            nxt=nums[row][col+1]

            heapq.heappush(heap,(nxt,row,col+1))

            max_val=max(max_val,nxt)

        return best
```

# 🧠 Pattern Recognition

If problem says:

```
merge k sorted lists
merge k sorted arrays
range from k lists
```

Trigger:

```
K WAY MERGE
```

---

### Mental Model

Think of **K sorted streams**:

```
stream1
stream2
stream3
...
streamK
```

Heap keeps **current smallest element of each stream**.
