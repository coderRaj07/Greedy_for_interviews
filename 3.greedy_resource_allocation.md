# 3️⃣ Greedy Resource Allocation

Core thinking:

You have **limited resource** and many **demands**.

Two typical greedy rules:

```
1️⃣ Satisfy smallest demand first
2️⃣ Pick largest reward first
```

---

# 1️⃣ Assign Cookies

[https://leetcode.com/problems/assign-cookies/](https://leetcode.com/problems/assign-cookies/)

### Problem

```
children greed = [g1,g2,g3]
cookies size = [s1,s2]
```

Child satisfied if:

```
cookie >= greed
```

Goal → **maximize satisfied children**

---

### Visual Reasoning

```
Child greed : 1 2 3
Cookies     : 1 1
```

If we give **big cookie to small greed first**, waste happens.

Correct greedy:

```
smallest greed → smallest cookie
```

---

### Algorithm

```
sort greed
sort cookies

if cookie >= greed
    assign
```

---

### Python

```python
class Solution:
    def findContentChildren(self, g, s):

        g.sort()
        s.sort()

        i=j=0
        count=0

        while i<len(g) and j<len(s):

            if s[j] >= g[i]:
                count+=1
                i+=1
                j+=1
            else:
                j+=1

        return count
```

# 2️⃣ Fractional Knapsack

[https://www.geeksforgeeks.org/problems/fractional-knapsack](https://www.geeksforgeeks.org/problems/fractional-knapsack)

### Problem

Items have:

```
value
weight
```

Bag capacity = W

But you **can take fraction**.

---

### Key Greedy Insight

Take item with **maximum value per weight**.

```
value / weight
```

Because each unit weight gives **max profit**.

---

### Example

```
value weight ratio
60     10     6
100    20     5
120    30     4
```

Pick **highest ratio first**.

---

### Python

```python
class Solution:
    def fractionalKnapsack(self, W,Items,n):

        Items.sort(key=lambda x:x.value/x.weight,reverse=True)

        total=0

        for item in Items:

            if W>=item.weight:

                total+=item.value
                W-=item.weight

            else:

                total+=item.value*(W/item.weight)
                break

        return total
```

# 3️⃣ Minimum Coins

[https://www.geeksforgeeks.org/problems/-minimum-number-of-coins](https://www.geeksforgeeks.org/problems/-minimum-number-of-coins)

Coins:

```
1 2 5 10 20 50 100 200 500 2000
```

Goal:

```
make amount with minimum coins
```

---

### Greedy Idea

Always take **largest coin first**.

Example

```
amount = 93
```

Take:

```
50
20
20
2
1
```

---

### Python

```python
def minCoins(amount):

    coins=[2000,500,200,100,50,20,10,5,2,1]

    res=[]

    for coin in coins:

        while amount>=coin:

            amount-=coin
            res.append(coin)

    return res
```

# 4️⃣ Gas Station

[https://leetcode.com/problems/gas-station/](https://leetcode.com/problems/gas-station/)

### Problem

Circular stations.

```
gas[i]
cost[i]
```

Return start index.

---

### Important Insight

If total gas < total cost → impossible.

Otherwise solution exists.

Greedy rule:

```
If tank becomes negative
start from next station
```

---

### Python

```python
class Solution:
    def canCompleteCircuit(self, gas, cost):

        if sum(gas) < sum(cost):
            return -1

        start=0
        tank=0

        for i in range(len(gas)):

            tank+=gas[i]-cost[i]

            if tank<0:

                start=i+1
                tank=0

        return start
```

# 5️⃣ Jump Game

[https://leetcode.com/problems/jump-game/](https://leetcode.com/problems/jump-game/)

### Problem

Each index tells **max jump length**.

Goal → reach end.

---

### Greedy Logic

Maintain **farthest reachable index**.

```
if i > farthest
     impossible
```

---

### Python

```python
class Solution:
    def canJump(self, nums):

        farthest=0

        for i in range(len(nums)):

            if i>farthest:
                return False

            farthest=max(farthest,i+nums[i])

        return True
```

# 6️⃣ Jump Game II

[https://leetcode.com/problems/jump-game-ii/](https://leetcode.com/problems/jump-game-ii/)

Goal → **minimum jumps**.

---

### Greedy Window Idea

Like **BFS levels**.

```
current_end
farthest
```

When index reaches `current_end`, we make jump.

---

### Python

```python
class Solution:
    def jump(self, nums):

        jumps=0
        cur_end=0
        farthest=0

        for i in range(len(nums)-1):

            farthest=max(farthest,i+nums[i])

            if i==cur_end:

                jumps+=1
                cur_end=farthest

        return jumps
```
